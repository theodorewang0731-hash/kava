## 🎯 选择建议

### 场景 1: AI 助手很智能（如 GPT-4o, Claude Sonnet）
**推荐：** `COPY_PASTE_PROMPT.txt` 或 `PROMPT_FOR_AI.txt`

简短的提示词足够，AI 能自主处理大部分问题。

---

### 场景 2: AI 助手一般（如 GPT-3.5, 基础模型）
**推荐：** `AI_ASSISTANT_PROMPT.md`

需要更详细的说明和上下文，避免 AI 理解错误。

---

### 场景 3: 首次使用 HPC 或遇到问题
**推荐：** `CONVERSATION_GUIDE.md`

按照对话脚本一步步进行，可以精确控制进度并应对各种情况。

---

### 场景 4: 快速测试
**推荐：** `COPY_PASTE_PROMPT.txt`

先快速尝试，如果遇到问题再升级到更详细的版本。

## 📋 使用流程

### 步骤 1: 选择合适的提示词文档

根据上面的建议选择一个版本。

### 步骤 2: 打开文档

```bash
# 在 HPC 上
cd ~/kava
cat <选择的文件名>

# 或在本地
# 打开对应的 .txt 或 .md 文件
```

### 步骤 3: 复制内容

- **超简版/快速版/完整版：** 复制全部内容
- **对话版：** 只复制第一轮对话的内容

### 步骤 4: 粘贴给 AI 助手

在 HPC 的 AI 助手界面粘贴内容。

### 步骤 5: 等待 AI 回复并跟进

- 如果使用超简版/快速版/完整版：AI 会开始执行
- 如果使用对话版：按照脚本进行下一轮对话

## 💡 提示词使用技巧

### 1. 分块执行（推荐）

不要一次给 AI 太多任务：

❌ 坏例子：
```
运行所有步骤 1-6
```

✅ 好例子：
```
先运行步骤 1，完成后告诉我结果
```

### 2. 明确期望

告诉 AI 你期望看到什么：

❌ 坏例子：
```
检查任务状态
```

✅ 好例子：
```
运行 squeue -u $USER
应该看到 12 个任务，状态为 RUNNING 或 PENDING
```

### 3. 要求完整输出

避免 AI 总结或省略信息：

❌ 坏例子：
```
告诉我结果
```

✅ 好例子：
```
把完整的输出复制给我，包括所有的 ✓ 和 ✗
```

### 4. 一次一步

执行复杂任务时：

❌ 坏例子：
```
配置环境并启动训练
```

✅ 好例子：
```
第一步：配置环境
运行：bash setup_hpc.sh
完成后告诉我。

（等 AI 完成后）

第二步：启动训练
运行：bash start.sh
```
